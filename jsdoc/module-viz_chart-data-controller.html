Description¶

This is a utility to manage charts’ experiment data in one global place and distribute to charts throughout UI. The mechanism for this is roughly diagrammed [here]{@link https://hms-dbmi.slack.com/files/alexkb/F4C8KQKMM/chartdatacontroller.png } .

DPROPERTIES

PROPS

Functions¶

initialize¶

This function must be called before this component is used anywhere else.

initialize(requestURLBase[, updateStats][, callback][, resync])
Arguments:
  • requestURLBase (string) – Where to request ‘all experiments’ from.
  • updateStats (function) – Callback for updating QuickInfoBar, for example, with current experiments, experiment_sets, and files counts.
  • callback (function) – Optional callback for after initializing.
  • resync (number|boolean) – How often to resync data, in ms, if window is active, for e.g. if submitters submitted new data while user is browsing.
Return undefined:
 

This function must be called before this component is used anywhere else.

isInitialized¶

Whether component has been initialized and may be used.

isInitialized()¶
Return boolean:True if initialized.

registerUpdateCallback¶

For React components to register an “update me” function, i.e. forceUpdate, to be called when new experiments/filteredExperiments has finished loading from back-end.

registerUpdateCallback(callback, uniqueID)¶
Arguments:
  • callback (function) – Function to be called upon loading ‘experiments’ or ‘all experiments’. If registering from a React component, should include this.forceUpdate() or this.setState(..).
  • uniqueID (string) – A unique identifier for the registered callback, to be used for removal or overwrites.
Return function:
 

A function which may be called to unregister the callback, in lieu of ChartDataController.unregisterUpdateCallback.

unregisterUpdateCallback¶

The opposite of registerUpdateCallback.

unregisterUpdateCallback(uniqueID)¶
Arguments:
  • uniqueID (string) – ID given alongside initially-registered callback.
Return undefined:
 

The opposite of registerUpdateCallback.

registerLoadStartCallback¶

Same as registerUpdateCallback but for when starting AJAX fetch of data.

registerLoadStartCallback(callback, uniqueID)¶
Arguments:
  • callback (function) – Function to be called upon starting AJAX load.
  • uniqueID (string) – @see ChartDataController.registerUpdateCallback().
Return function:
 

Function for unregistering callback which may be used in lieu of @see ChartDataController.unregisterUpdateCallback().

unregisterLoadStartCallback¶

The opposite of registerLoadStartCallback.

unregisterLoadStartCallback(uniqueID)¶
Arguments:
  • uniqueID (string) – ID given alongside initially-registered callback.

getState¶

Get current state. Similar to Redux’s store.getState().

getState()¶

setState¶

Analogous to a component’s setState. Updates the private state of ChartDataController and notifies update callbacks if experiments or filtered experiments have changed.

setState(updatedState[, callback])¶
Arguments:
  • updatedState (Object) – New or updated state object to save.
  • callback (function) – Optional callback function.
Return *:

Result of callback, or undefined.

sync¶

sync([callback])
Arguments:
  • callback (function) – Function to be called after sync complete.
Return undefined:

updateStats¶

Update stats in top left corner of page, if updateState param was passed in during initialization.

updateStats()¶
