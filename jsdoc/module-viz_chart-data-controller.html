Static Public initialize¶

This function must be called before this component is used anywhere else.

initialize(requestURLBase[, updateStats][, callback][, resync])¶
Arguments:
  • requestURLBase (string) – Where to request ‘all experiments’ from.
  • updateStats (function) – Callback for updating QuickInfoBar, for example, with current experiments, experiment_sets, and files counts.
  • callback (function) – Optional callback for after initializing.
  • resync (number|boolean) – How often to resync data, in ms, if window is active, for e.g. if submitters submitted new data while user is browsing.
Returns undefined:

Static Public registerUpdateCallback¶

For React components to register an “update me” function, i.e. forceUpdate, to be called when new experiments/filteredExperiments has finished loading from back-end.

registerUpdateCallback(callback, uniqueID)¶
Arguments:
  • callback (function) – Function to be called upon loading ‘experiments’ or ‘all experiments’. If registering from a React component, should include this.forceUpdate() or this.setState(..).
  • uniqueID (string) – A unique identifier for the registered callback, to be used for removal or overwrites.
Returns function:
 

A function which may be called to unregister the callback, in lieu of ChartDataController.unregisterUpdateCallback.

Internally used to generate a URL from current href and expSetFilters from Redux store to fetch filtered/selected experiments from back-end. If no ‘search’-compatible href is set in Redux store, ‘/browse/’ is used.

  • href (string) – Current href from Redux store.
