Component: viz/BarPlotChart¶

Visualization component for the BarPlot. Contains chart and labels only – no controls. To add controls, wrap the chart in BarPlotChart.UIControlsWrapper, which will feed its state as props to BarPlotChart and has UI components for adjusting its state to select Charting options. Use BarPlotChart (or UIControlsWrapper, if is wrapping BarPlotChart) as child of ChartDataController.provider, which will feed props.experiments and props.filteredExperiments.

Props¶

experiments¶

Type: Array.<Object>

List of all experiments as stored in and provided by ChartDataController.

filteredExperiments¶

Type: Array.<Object>

List of experiments which match current filters. Stored in and provided by ChartDataController[.Provider].

onBarPartMouseEnter¶

Type: function

A callback function for when someone’s cursor enters a bar part. Takes the node/datum of the bar part (0) and MouseEvent (1) as arguments.

onBarPartMouseLeave¶

Type: function

Counterpart for props.onBarPartMouseEnter.

fields¶

Type: Array.<Object>

List of field objects containing ‘field’[, ‘name’][, ‘description’][, ‘title’]. If length === 1, only plots bars (no bar parts), if length === 2, plots 2nd field as subdivision. Provide more along with props.useOnlyPopulatedFields = true to have chart auto-select the fields to plot based on which has than 1 term.

useOnlyPopulatedFields¶

Type: boolean

Defaults to false. If true, and list of fields is longer than 2, will visualize first field(s) found in list with more than 1 term.

width¶

Type: number

Self explanatory.

height¶

Type: number

Self explanatory.

aggregateType¶

Type: string

Set by UIControlsWrapper. Controls whether Y-Axis has ‘experiment_sets’, ‘experiments’, or ‘files’.

showType¶

Type: string

Set by UIControlsWrapper. Controls whether showing “all” experiments or only the selected or “filtered”-in experiments.

Public Static genChartBarDims¶

Return an object containing bar dimensions for first field which has more than 1 possible term, index of field used, and all fields passed originally.

genChartBarDims(fields, fields.terms, fields.total[, availWidth][, availHeight][, styleOpts][, useOnlyPopulatedFields][, maxValue])¶
Arguments:
  • fields (Array.<Object>) – Array of fields (i.e. from props.fields) which contain counts by term and total added through @see aggregationFxn.genChartData().
  • fields.terms (Object) – Object keyed by possible term for field, with value being count of term occurences in [props.]experiments passed to genChartData.
  • fields.total (number) – Count of total experiments for which this field is applicable.
  • availWidth (number) – Available width, in pixels, for chart.
  • availHeight (number) – Available width, in pixels, for chart.
  • styleOpts (Object) – Style settings for chart which may contain chart offsets (for axes).
  • useOnlyPopulatedFields (boolean) – Determine which fields to show via checking for which fields have multiple terms present.
  • maxValue (number) – Maximum y-axis value. Overrides height of bars.
Returns Object:

Object containing bar dimensions for first field which has more than 1 possible term, index of field used, and all fields passed originally.
